自然数の足し算の定義がwell-definedであることを確かめる 交換法則編

先週末、東京に行っておりましたが、旅の途中で考えていた話をまとめておきます。

どうにも、バスの中でうまく眠れず、ここ最近考えていた、足し算の交換法則について考えていました。 そもそも、なんで考えていたのかということには複雑な理由があるのですが、それは察してくださいということで。 疑問としては、『一階述語論理、ペアノ公理を前提(公理)として、原子再帰的関数として与えられた自然数の足し算がwell-defined、つまり、僕らが日常で素朴な数え上げ等で認識する足し算の満たす性質を満たしているのか』ということです。

さて、以下、証明に関するメモです。メモと書いているのは、自然言語を使っているからですね。

足し算は、　任意の自然数n,mについて

  • n + 0 = n
  • n + suc(m) = suc(n + m)

と定義します。 sucは後者関数(つまりsuc(n)はｎの後者)として、0はどの自然数の後者にもならない自然数です。

この定義より、任意の自然数nについて、

ｎ + suc(0) = suc(n + 0) = suc(n) = suc(n) + 0

となることが、言えます。 次に、自然数mについて、

任意の自然数ｎについて、 n + suc(m) = suc(n) + m

ならば、

任意の自然数nについて、 n + suc(suc(m)) = suc(n + suc(m)) = suc(suc(n) + m) = suc(n) + suc(m)

であるので、ペアノ公理(数学的帰納法)より、任意の自然数について、

n + suc(m) = suc(n) + m

が言えます。あとで、これを使うので、補題Iとしておきましょう。

次に、足し算の定義より、

0 + 0 = 0

また、

n + 0 = 0 + n

ならば、

suc(n) + 0 = suc(n) = suc(n + 0) = suc(0 + n) = 0 + suc(n)

であるので、任意の自然数ｎについて、

n + 0 = 0 + n

が言えます。さらに、自然数mについて、

任意の自然数nについて、 n + m = m + n

ならば、

任意の自然数nについて、 n + suc(m) = suc(n + m) = suc(m + n) = m + suc(n) = suc(m) + n

です。 ここで、補題Iを使いました。 このため、数学的帰納法より、

任意の自然数n, mについて

n + m = m + n

が言えました。

つまり、最初の足し算の定義は、素朴なあるいはモノイドとしての自然数の足し算の性質のうち交換するということをちゃんと満たしているということがわかりました。

計算間違い等があれば教えてください。

ではまた。
